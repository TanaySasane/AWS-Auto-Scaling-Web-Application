Auto Scaling Group

Maintains desired capacity across Availability Zones using scaling policies triggered by CPU, RPS, or custom CloudWatch metrics.

Self-healing replaces failed instances automatically.

Application Load Balancer

Routes HTTP/HTTPS traffic across multiple EC2 targets while performing health checks.

Supports content-based routing for blue/green deployments.

VPC & Networking

Public subnets host ALB; private subnets keep EC2 instances isolated.

Security groups control ports, while route tables enforce isolation.

Monitoring & Alerts

CloudWatch collects metrics (CPU, latency, requests) that feed Auto Scaling decisions.

Alarms trigger SNS notifications and can be tied to Lambda or Step Functions.
